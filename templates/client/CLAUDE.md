# __PRODUCT__-client — agent guide

The __PRODUCT__ product frontend (Vite + React + TypeScript), stamped from the
memQL project template. Sibling of `../__PRODUCT__-carrier` (the product BFF +
DSL) and `../memql` (the shared engine), consolidated by the workspace `go.work`.

## Golden rule: bare ids

Canonicalization is **server-side only**. This client uses **bare short-slug ids
everywhere** — it never composes, parses, or compares canonical `v1:` ids.

- The ONLY place canonical ids live is `src/generated/concepts.ts` (generated by
  the carrier's `make sdk-gen`; the DSL source of truth). Refer to concepts
  THROUGH its exports (`Concepts` / `CDCTopics` / `CDCFilters` / `topicFor` /
  `filterFor`), never a hand-written literal.
- `eslint.config.js` enforces this: `v1:` literals and `.split(':')` /
  `.lastIndexOf(':')` id surgery are lint errors. Run `npm run lint`.
- When concepts mix, key rows by `(concept, id)` via `nodeKey`, never by id alone.

## Layout

- `src/lib/memql/client.ts` — the memQL client (thin local WS client in the
  starter; swap for `@__PRODUCT_ORG__/__PRODUCT__-sdk`'s `Connection`/`QueryClient`
  once published — see README "SDK resolution").
- `src/lib/auth/identity.ts` — magic-link login against `https://identity.__DOMAIN__`.
- `src/context/Session.tsx` — session + client provider (`useSession` / `useMemql`).
- `src/generated/concepts.ts` — generated bare-ids concept/topic constants.
- `src/pages/` — `Login` (magic-link), `Home` (one query round-trip), `Live`
  (one CDC subscription via generated filter constants).

## SDK

The production SDK is `@__PRODUCT_ORG__/__PRODUCT__-sdk` (GitHub Packages),
generated by the carrier's `make sdk-gen`. The starter shell is self-contained
(no SDK dependency) so it builds at stamp time; see README "SDK resolution" for
the wiring steps + the `NODE_AUTH_TOKEN` requirement once you consume the
published package.
